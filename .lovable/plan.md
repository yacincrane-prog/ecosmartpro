

# إضافة إمكانية تعديل وحفظ البيانات الناقصة للمنتجات المزامنة

## المشكلة
منتجات مثل "Adap mic" و "Jmary KP2209" تصل من EcoSmart بـ `sale_price = 0` — بيانات ناقصة. الحقول حالياً `ReadOnly` ولا يمكن تعديلها، فيحسب النظام خسارة وهمية.

## الحل
تحويل الحقول الحرجة (سعر البيع، سعر الشراء، تخفيض التوصيل) إلى حقول قابلة للتعديل مع حفظ التعديلات في قاعدة البيانات.

---

## التنفيذ

### 1. توسيع جدول `synced_product_inputs` (Migration)
إضافة أعمدة جديدة لحفظ التعديلات اليدوية:
- `sale_price_override` (numeric, nullable)
- `purchase_price_override` (numeric, nullable)
- `delivery_discount_override` (numeric, nullable)

### 2. تحديث `useSyncStore.ts`
- توسيع `ProductManualInputs` لتشمل الأعمدة الجديدة
- تحديث `fetchManualInputs` و `saveManualInput` لدعم الحقول الجديدة

### 3. تحديث `SyncedDataPage.tsx`
- إضافة `drafts` state محلي لتتبع التعديلات قبل الحفظ
- دالة `getV(product, field)` تُرجع: القيمة المعدّلة → أو القيمة المحفوظة → أو القيمة المزامنة
- تحويل حقول سعر البيع وسعر الشراء وتخفيض التوصيل من `ReadOnlyField` إلى `Input` قابل للتعديل
- إضافة زر **"حفظ التعديلات"** يظهر فقط عندما يوجد تغيير
- إضافة مؤشر بصري (أيقونة تعديل) للحقول المعدّلة يدوياً لتمييزها عن القيم المزامنة
- استخدام القيم المعدّلة في جميع الحسابات (revenue, profit, etc.)

### 4. UX
- الحقول المزامنة بقيمة صحيحة: تظهر عادية مع إمكانية التعديل
- الحقول بقيمة 0 (ناقصة): تظهر بحدود صفراء تنبيهية + placeholder يقول "أدخل القيمة"
- بعد التعديل: حدود زرقاء تدل على أن القيمة معدّلة يدوياً
- زر "إعادة تعيين" لإرجاع القيمة للمزامنة الأصلية

## الملفات المتأثرة

| ملف | تغيير |
|-----|--------|
| Migration جديدة | إضافة أعمدة override |
| `src/store/useSyncStore.ts` | دعم الحقول الجديدة |
| `src/pages/SyncedDataPage.tsx` | حقول قابلة للتعديل + حفظ + حسابات محدّثة |
